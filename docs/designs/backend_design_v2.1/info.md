一、表格高度 / 滚动策略
短答：全站统一一种范式——「页面滚动 = 顶栏 + 工具栏；表格内滚动 = 数据 + 表头粘性 + 分页底栏」。不同数据量靠分页策略切换来吸收差异，不靠"哪种页面不同滚法"。
推荐方案：表格内独立滚动 + 表头粘性（默认）
布局拆三层：
┌─ 顶栏（固定）
├─ 工具栏 / Tab / 筛选条（页面级，跟随页面滚动到顶后吸附）
└─ 表格容器（剩余高度，flex:1, min-height:0）
├─ thead（sticky top:0）
├─ tbody（overflow:auto，承载滚动）
└─ pagination + 批量浮条（sticky bottom:0）
好处：

- 表头永远可见，列对齐稳定
- 批量浮条 / 分页栏永远可见
- 工具栏 / 筛选条不会被表格的滚动条挤压
- 多 Tab 页（如内容审核）切 Tab 时滚动位置独立
  实现要点：表格容器外层用 display:flex; flex-direction:column; height:calc(100vh - topbar - workspaceTopPad)，tbody 区 flex:1; min-height:0; overflow:auto。这是一处统一的 hook，所有列表页复用。
  例外情况（少数）
- 超短列表（≤ 10 行）：直接 auto height，无需滚动条；分页栏紧贴最后一行。
- 嵌套展开行展开后内容很长（如采集站点展开包含子表格 + 日志）：展开区在表格内是"行高扩展"，外层仍走表格滚动。子区如果太高，子区内部再加自己的滚动（罕见）。
- 审核三栏布局：左队列、中预览、右元信息——三栏各自滚动，不走表格范式。
  分页 vs 滚动 — 统一策略
  按数据量分三档（用同一个组件，参数切换）：
  数据量 翻页方式 视觉表现
  < 200 条 客户端分页（每页 50/100） 普通页码栏
  200 – 50,000 条 服务端分页（每页 20/50/100） 页码栏 + 总数 + 跳转
  > 50,000 条 游标分页 + 虚拟滚动 "前 N / 后 N"按钮 + 跳转禁用，显示"已加载 X 条"
  > 切换是组件内部的事，用户层面唯一变化是大数据集没有"跳到第 N 页"——这是无法避免的物理约束，但 99% 的列表页 < 50k，跳页正常工作。
  > 统一一处：一个 <DataTable> 组件，prop paginationMode="auto" 根据 totalCount 自动选档。开发只关心 schema + fetcher。
  > 我会在 hi-fi 阶段把这套范式写成一个标杆页（视频库），再扩展到其他列表。

二、采集站点展开显示线路 + 线路别名
完全同意。建议在站点展开区域内新加一个"线路"分组，与"分类映射"、"采集配置"并列：
站点展开区
├── 分类映射
├── 线路 / 别名 ← 新增
├── 采集配置（MACCMS）
├── 最近采集
└── 健康度
线路区内：
列 说明
原始名（来源给的） 灰色弱化、等宽字体（往往是 dplayer / m3u8_8 这类）
别名 可就地编辑，留空时回退显示原名
协议 / 容器 m3u8 / mp4 / flv
Probe HEAD 探测状态（沿用 dual-signal）
Render 实际播放渲染状态
视频数 该线路绑定的视频数
操作 测试播放 · 禁用 · 合并到…
补充建议：

- 别名"全局映射"：在"播放线路"页提供一张全局别名表（原名 → 显示名 + 跨站点共享），避免每个站点重复改。站点展开时下拉自动建议匹配。
- 批量改名：选中多条线路 → 批量设别名（支持正则替换，如 ^dpl(\d+)$ → 备线$1）
- 别名优先级：站点级别名 > 全局别名 > 原名（站点级覆盖全局）
- 冲突警告：同一站点内若两条线路别名相同 → 红字警告，不允许保存
  我会把"线路"区作为站点展开默认折叠的第二个分组（"分类映射"展开默认显示，"线路"按需展开），免得展开区一次占太高。
