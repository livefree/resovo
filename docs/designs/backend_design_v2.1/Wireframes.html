无刷新策略（关键）
  • 状态切换：先改本地 row state，再发请求；失败回滚 + Toast
  • 表头排序/过滤变更：仅 tbody 重排，header 不闪
  • 分页：保留 thead；tbody 渐隐 → 渐显（150ms）
  • 新增行：插入到顶部，蓝高亮 1.5s 后淡出
  • 删除行：行内淡出 + 折叠高度，无整表 reflow
高度与滚动策略 · 全站统一

一种范式：页面滚动止于工具栏；表格容器占剩余高度，thead 粘顶 / tbody 滚动 / 分页栏粘底。批量浮条粘在 tbody 与分页之间，永远可见。

顶栏（fixed）
工具栏 / 筛选 / Tab（页面滚到顶后 sticky）
▣ thead · sticky top:0
tbody · overflow:auto（滚动发生在这）
▣ 批量浮条 · sticky bottom（仅选中时出现）
▣ 分页 · sticky bottom:0
外层 flex column · height: calc(100vh − topbar) · tbody flex:1 / min-height:0 / overflow:auto
分页档位（同一组件，参数切档）
数据量 翻页方式 视觉
< 200 客户端分页 50/100 页码栏
200 – 50,000 服务端分页 20/50/100 页码 + 跳转 + 总数
> 50,000 游标 + 虚拟滚动 前 N / 后 N，禁跳页
DataTable 组件 paginationMode="auto"，按 totalCount 自动选档，使用方无感。
例外
  • 列表 ≤ 10 行 → auto 高度，无 tbody 滚动
  • 展开行内容很长 → 行高扩展走外层滚动；子区超长再嵌套
  • 三栏布局（审核）→ 三栏各自滚动，不走表格范式
无刷新（再强调）
  • 翻页 / 排序 / 过滤变更 → 仅 tbody fade，thead / 工具栏不闪

弹层规范 · Modal / Drawer / Popover · 三选一

DECISION RULE

统一三种弹层语义。混用是当前最大的体验隐患；下表是接下来所有弹层 PR 的判断依据。

Modal · 中央阻塞
居中弹窗
  • 危险确认（删除 / 全量采集 / 改判）
  • 关键单步表单（< 5 字段）
  • 必须用户响应才能继续
Drawer · 右侧滑出 · 半阻塞
  • 详情查看 / 长表单（视频编辑 / 站点配置）
  • 需要保留上下文（左侧列表仍可见）
  • 支持上一个 / 下一个 切换
Popover · 就近 · 非阻塞
就近浮层
  • 表头 ⋯ 菜单 / 日期选择 / 颜色选择
  • 快捷过滤 / 单字段编辑
  • 点外部即关闭，无遮罩
判断流程
  1. 这次操作需要用户全神贯注响应吗？是 → Modal
  2. 否，那这次操作需要看到背景上下文吗？是 → Drawer / Popover
  3. 表单 > 5 字段 / 多 Tab / 需要预览？→ Drawer
  4. 否则 → Popover
