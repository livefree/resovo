## 1. 这个页面是做什么的

采集控制中心。从外部源站抓取视频元数据与播放线路；监控每个站点的运行状态、最近任务、健康度；触发增量/全量采集。**「采集→审核→上架」金票工作流的起点**（[W1](../10-workflows/W1-crawl-to-publish.md)）。

## 3. 常用操作

### 3.1 触发全站采集（CHG-SN-8-01 修订）

> **重要变更**（2026-05-21）：「全站全量」从主按钮位置**移除**，移入「高级 ▾」dropdown 并加双重确认；常规位置为「全站增量」（默认主按钮）。

#### 3.1.1 触发「全站增量」（推荐 / 高频）
- **前置条件**：crawler/admin 角色；当前 `freezeEnabled=false`
- **操作步骤**：
  1. 点 PageHeader 右上「全站增量」（蓝色主按钮）
  2. 单次确认弹窗：「确定对全站发起增量采集？」→ 确定
- **期望结果**：toast "已发起全站增量 · runId=xxxxxxxx · 入队 N 个站点"；TimelineCard 立即显示新任务
- **失败处理**：
  - toast "采集已冻结" → 先去「高级 ▾」→ 「解除冻结」
  - toast "操作冲突 409" → 已有 run-all 在进行中，等 1 分钟重试

#### 3.1.2 触发「全站全量采集」（不推荐 / 低频危险）
> ⚠️ **全量重采约等于全站重建索引；正常运营不需要全量。**

- **前置条件**：admin 角色（更严格）；当前无 run-all 运行中
- **操作步骤**：
  1. 点 PageHeader「高级 ▾」→ 展开
  2. 点最顶项「全站全量采集」（红色，danger 样式）
  3. **第一次确认**：「确定对全站发起**全量**采集？此操作会创建多个 task。」→ 确定
  4. **第二次确认**（输入校验）：弹窗要求输入「全量」二字以继续；输错任意非「全量」字符串 → 中止
- **期望结果**：toast "已发起全站全量"；TimelineCard 多任务排队
- **失败处理**：
  - 第二次确认输错 → 静默中止（不报错，再次操作即可）
  - 其余错误同 3.1.1

### 3.2 站点级操作（CHG-SN-8-02 填写）

#### 3.2.1 站点行触发增量 / 全量
- **位置**：站点列表行尾「操作」列内 inline btn「+ 增量」「+ 全量」+「⋯」dropdown
- **前置条件**：站点 `disabled=false` + 全局 `freezeEnabled=false`
- **操作**：直接点对应 btn 触发该站点的增量 / 全量采集
- **期望结果**：toast「已发起增量 · jszyapi · runId=xxxxxxxx」；表中该行的「最近采集」列状态 pill 自动刷新为「运行中」（如 KPI poll 命中）

#### 3.2.2 读懂「最近采集」列（CHG-SN-8-02 新增 status pill）
站点列表「最近采集」列由两部分组成：
- **上行 status pill**：「成功」（绿）/「失败」（红）/「运行中」（蓝）/「未采集」（灰）
- **下行相对时间**：「5 分钟前」/「2 小时前」/「3 天前」/「—」（无 lastCrawledAt）

> 旧版本只显示时间无法立刻看出成功/失败，CHG-SN-8-02 起补 pill 显式区分。

#### 3.2.3 站点行展开
- 点击行首 chevron `›` 展开 → 显示线路矩阵（来自 sources 域）+ 分类映射
- 二次点击折叠

### 3.3 采集完成深链至审核台（CHG-SN-8-03 / W1 ②）

- 触发「全站增量」或「全站全量采集」成功后，toast 上方除了基本信息（runId + 入队站点数）外，**右侧含「查看本次新增视频」action 按钮**
- 点击 → 自动跳 `/admin/moderation?run_id=<runId>`
- 进审核台后顶部显示 banner「来自采集 run xxxxxxxx · 新增视频按创建时间排在队列顶部；逐条 J/K 翻页处理」
- 「清除筛选」按钮可移除该 run_id 参数（保留其它过滤）
- **软深链说明**：本期未在后端按 runId 严格过滤队列（后端 schema 修订属 ADR 协议范围，推迟到 CHG-SN-8-03-B）；banner 是 UI 提示，提醒运营来自哪次 run；新增视频自然按 `createdAt desc` 排在队列顶部

## 4. 进阶操作（低频 / 危险）

### 4.1 全局止血（destructive）
- **影响**：开启采集冻结 + 取消所有待处理任务 + 信号现有运行任务终止
- **回滚**：到「高级 ▾ → 解除冻结」恢复；已 cancel 的 task 不可恢复
- **操作**：高级 ▾ → 「全局止血」 → 双重 confirm（第二次按钮显示文字）

### 4.2 重建 ES 索引（低频运维）
- **影响**：搜索索引重建（耗时较长）；运行期间搜索结果可能临时不准
- **回滚**：无（自然完成）
- **操作**：高级 ▾ → 「重建 ES 索引」 → 双重 confirm

### 4.3 开启/解除全局冻结
- 影响：所有采集 / 重探 / 行级写操作被拒绝
- 操作：高级 ▾ → 「开启冻结」/「解除冻结」 → 单次 confirm

## 5. 字段含义

| 字段 | 含义 | 取值 | 排序 | 筛选 |

## 6. 状态颜色

「最近采集」status pill 颜色矩阵：

| 颜色 | status | 含义 |
|---|---|---|
| 绿（ok）| `ok` | 上次采集成功 |
| 蓝（info）| `running` | 正在采集中 |
| 红（danger）| `failed` | 上次采集失败，需要关注 |
| 灰（muted）| `null` | 从未采集（新站点）|

「健康度」progress bar 颜色：≥80 绿 / 40-79 黄 / <40 红。

时间轴 pill 颜色：实时 / 暂停 / 全局冻结（三态切换）。

## 7. FAQ / 常见错误

| 现象 | 原因 | 解决方法 |
|---|---|---|
| 触发采集后 toast「采集已冻结」 | 全局 freeze 开启 | 「高级 ▾ → 解除冻结」 |
| 触发增量 toast「操作冲突 409」 | 已有 run-all 在进行 | 等候 1-2 分钟后重试 |
| 最近采集 status 显示「失败」 | 上次采集报错 | 查 [`/admin/crawler/runs`](./P-crawler.md) 看 task 日志 |
| 站点行级「增量」按钮 disabled | 站点 `disabled=true` 或全局冻结 | 「⋯ → 启用」/ 解除冻结 |
