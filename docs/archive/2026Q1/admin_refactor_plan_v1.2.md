## 一、阶段目标

在 v1.1（CHG-54 ~ CHG-59）完成基础上，本阶段目标：

- 完成 `crawler-site` 的纵向闭环拆分（容器、工具、视图职责清晰）
- 完成 `system` 目录业务归组，消除根目录平铺
- 明确页面入口（容器）与业务模块（组件/hooks/utils）边界
- 建立可复用的模块结构模板（仅约束新增代码，不强制一次性改完旧代码）
- 为下一阶段 shared 抽象与 UI/UX 优化准备输入

------------------------------------------------------------------------

## 二、执行原则

1. 行为不变（硬约束）
2. 只做结构，不做逻辑升级
3. 优先纵向切分，不提前做横向抽象
4. 单任务可回滚（一个任务一个 commit）
5. 每步可验证（自动化 + 手动关键路径）
6. 禁止顺手改无关文件

------------------------------------------------------------------------

## 三、项目现状校准（基于当前仓库）

当前已完成：

- `crawler-site` 已有：`tableState`、`importParser`、`useCrawlerSiteColumns`、`useCrawlerSiteSelection`、`CrawlerSiteTable`
- `config-file` 已有：`constants`、`utils`

当前命令口径（必须统一使用）：

- `npm run typecheck`
- `npm run lint`
- `npm run test:run -- <specs>`

说明：计划文档后续不再使用 `pnpm` 作为验收命令，避免执行口径漂移。

------------------------------------------------------------------------

## 四、阶段任务列表（按执行顺序）

### 任务1：crawler-site 纵向闭环补齐（最高优先级）

目标：将 `CrawlerSiteManager` 进一步收敛为容器，减少 UI/状态细节耦合。  
约束：不改 API、不改字段、不改用户可见交互。

拆分为 4 个原子子任务（每个单独 commit）：

1. `CrawlerSiteToolbar` 组件抽离
2. `CrawlerSiteFilters` 组件抽离
3. `CrawlerSiteFormDialog`（新增/编辑）抽离
4. `useCrawlerSites`（列表加载与刷新）抽离

验收：

- 列显隐/排序/筛选持久化行为不变
- 列宽拖拽与批量操作行为不变
- `CrawlerSiteManager` 仅保留编排职责

------------------------------------------------------------------------

### 任务2：system 目录业务归组（结构归位）

目标：消除 `system` 根目录平铺，统一目录语义。

目标结构：

- `system/crawler-site/*`
- `system/config-file/*`
- `system/monitoring/*`
- `system/migration/*`
- `system/site-settings/*`

说明：允许改 import/test path；禁止改业务行为。

验收：

- 路由可正常访问
- import 全部可解析
- 相关单测路径更新后通过

------------------------------------------------------------------------

### 任务3：页面入口与业务模块分离

目标：入口组件只负责容器职责，不承载具体业务实现细节。

验收：

- 入口组件仅包含：数据注入、权限、布局、模块装配
- 业务逻辑全部下沉至对应模块目录

------------------------------------------------------------------------

### 任务4：模块结构模板落地（增量约束）

统一模板（针对新改动必须遵守）：

```text
module/
  components/
    XxxTable.tsx
    XxxFilters.tsx
    XxxActions.tsx
  hooks/
    useXxxList.ts
    useXxxFilters.ts
  utils/
    *.ts
  types.ts
```

说明：旧代码不做一次性强迁移；在后续任务中逐步收敛。

------------------------------------------------------------------------

### 任务5：shared 抽象（延后，不在本阶段执行）

触发条件：同一能力被 >= 2 个模块稳定复用，且接口已稳定。

候选：

- Modal
- DataTable
- 通用 Form 控件

------------------------------------------------------------------------

### 任务6：UI/UX 重构准备（仅采样，不改 UI）

本阶段只做：

- 标记问题区域
- 收集截图与场景
- 问题归类（布局/交互/可读性/一致性）

------------------------------------------------------------------------

### 任务7：统一验收与质量保障

每个子任务完成时必须执行：

- `npm run typecheck`
- `npm run lint`
- `npm run test:run -- <受影响测试>`

每个阶段完成时必须手动验证：

- 列显隐
- 排序/筛选持久化
- 拖拽列宽
- 导入兼容
- 批量操作
- 配置文件“保存并同步”
