## 冻结范围

**冻结期间（M0–M6）禁止**接受以下类型的工作：
- 与三份方案目标无关的新业务功能
  - 方案 1：`docs/design_system_plan_20260418.md`（设计系统 / Token 层）
  - 方案 2：`docs/frontend_redesign_plan_20260418.md`（前端重写）
  - 方案 3：`docs/image_pipeline_plan_20260418.md`（图片管线）
- 对现有 API 接口的非必要扩展
- 后台管理新模块（与重写无关）
- 爬虫 / 元数据层新功能

## 例外（可在冻结期内接受）

以下情形为 **P0 例外**，可绕过冻结限制：

| 类型 | 定义 | 审批方式 |
|------|------|---------|
| 生产安全漏洞 | CVSS ≥ 7.0 的安全问题 | 直接修复，changelog 标注 P0-SEC |
| 数据损坏 bug | 导致用户数据写入错误或丢失 | 直接修复，changelog 标注 P0-DATA |
| 服务不可用 | 主要功能（播放、搜索）完全不可用 > 30 分钟 | 直接修复，changelog 标注 P0-SRV |

P0 修复必须是最小范围 hotfix，不得趁机附带其他改动。

冻结期收到的非 P0 需求，统一暂存到 `docs/task-queue.md` 末尾的"冻结期积压"区：

```markdown

```
