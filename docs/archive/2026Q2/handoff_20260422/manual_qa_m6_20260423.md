## 快速开始

### 启动本地 dev（三个进程）

```bash
# 1. API（港口 4000，multipart 已升 5MB）
npm run dev -w @resovo/api

# 2. web-next（港口 3002）
npm run dev -w @resovo/web-next

# 3. server（后台，港口 3001）
npm run dev -w @resovo/server
```

### 决定 R2 配置路径

- **路径 A — 用 R2 生产**（需 Cloudflare 账号）：
  `.env.local` 配齐 `R2_ENDPOINT` + `R2_ACCESS_KEY_ID` + `R2_SECRET_ACCESS_KEY` + `R2_IMAGES_BUCKET=resovo-images` + `R2_PUBLIC_BASE_URL=https://pub-xxx.r2.dev`
- **路径 B — 用本地 FS fallback**（推荐首次 QA）：
  不配 R2；可选加 `LOCAL_UPLOAD_DIR=.uploads` + `LOCAL_UPLOAD_PUBLIC_URL=http://localhost:4000/v1/uploads`

## Checklist（18 项，对应对齐表 §5）

### §5.1 CDN loader 链路（3 项）

1. 不设 IMAGE_LOADER → 访问 `/en/dev/fallback-preview` → 右侧 "next 模式" 区域 `<img src>` = 原 picsum URL
2. 设 `NEXT_PUBLIC_IMAGE_LOADER=cloudflare` + `NEXT_PUBLIC_IMAGE_LOADER_CF_ACCOUNT_HASH=test` → 重启 dev server → 同页面 `<img src>` 含 `imagedelivery.net/test/` + `w=` + `f=auto`
3. 回切 env → src 原样

### §5.2 视频图片上传（7 项）

选一个编辑中的视频（任何 published 或 pending_review 的），进入 `/admin/videos/{id}/edit`：

4. 四 kind 都显示"上传新图"按钮
5. 选合法图（> 100KB 以便看清进度）→ 按钮文案"上传中 XX%" + 进度条同步
6. 上传成功 → 缩略图更新 → 状态"检测中…" → 几秒后变"正常"
7. 尝试 > 5MB 图 → 显示"图片超过 5MB"
8. 尝试 .pdf → 显示"仅支持 JPEG..."
9. 点缩略图 → `<dialog>` 打开显示原图 + ESC 关闭 + 点遮罩关闭
10. "改 URL" 按钮填外链 → 保存 → 同样触发健康检查

### §5.3 Banner 图片上传（4 项）

11. 新建 Banner 页 `/admin/banners/new` → **无**"上传新图"按钮 + 显示"新建 Banner 时需先填写外链..."
12. 编辑 Banner 页 → 有"上传新图"
13. 上传新图 → 进度百分比 + 预览按 16:9 更新
14. 点缩略图放大 → dialog 比视频放大更宽（1200px max）

### §5.4 R2 未配场景（仅路径 B 验证；3 项）

15. 不配 R2 → 上传成功 → 返回 URL 形如 `http://localhost:4000/v1/uploads/posters/...png`
16. `apps/api/.uploads/posters/` 目录下有真实文件
17. 浏览器直接请求上述 URL → 返回图片

### §5.5 字体回归（CHORE-08，1 项）

18. DevTools > Network → 看到 `_next/static/media/*.woff2` 加载 Noto Sans / SC；body computedStyle `font-family` 含 `"Noto Sans SC"`
